# Project Summary: Malaysia Automobile Business Data Scraper

## Purpose of this File
This file serves as a persistent context tracker for this project. It is meant to be read and appended by different AI agents to maintain continuity, understand the project goals, and keep track of discussions, decisions, and changes made to the codebase.

## Project Description
A data scraping tool targeting businesses in Malaysia. The user inputs a "city", "pincode", and "number of businesses". The tool outputs well-framed profiles of businesses in that area, including:
- Business name
- Contact number
- Email
- Website link
- Links to accounts on other car-selling marketplace websites
- Address
- Reviews
- Additional parameters (to be determined)

## Phases
The project is divided into 3 phases.
**Phase 1:** Build a system using SerpAPI to retrieve a list of businesses in the given city and pincode via Google Maps. (Now integrated into the FastAPI backend via `search_logic.py`)
**Phase 2 (Current):** Enrich dealer profiles. 
   - Uses Jina AI API to search for the dealer's website, marketplace profiles (Mudah, Carlist, Autocari), and social media (Facebook, Instagram, Tiktok), calculating completeness scores.
   - Uses `EmailLinkedin.py` to extract emails and additional phone numbers from websites (with Firecrawl and Hunter.io fallbacks) and searches for LinkedIn company profiles via DuckDuckGo.
**Phase 3:** (To be defined) - Will add layers on top of Phase 1 & 2.

## Architecture
- **Backend:** A FastAPI server (`main.py`) that manages search/enrichment jobs asynchronously using threading and a job manager.
- **Frontend:** A web-based UI (`index.html`) that polls the backend for live status updates, logs, and displays the enriched dealer profiles, replacing the original CLI.
